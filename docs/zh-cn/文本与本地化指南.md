## 文本与本地化指南

本文档是一个工作草案。最终版本将位于 UI 系统的仓库中。

### 概述

熟悉 Victoria 2 modding 的人会以前就遇到过本地化 .csv 文件。每一行都包含一个键，后面跟着不同语言的条目。当游戏需要在用户界面中显示文本时，它使用键来查找该文本（并在此时选择合适的语言）。

新的 UI 系统将使用概念上类似的东西，尽管实现方式不同。我们的系统也将依赖于一组定义各种键的文本文件。然而，对于我们来说，每个文件将只包含一种语言，切换显示的语言将意味着切换到不同的一组文件。

### 概念要点：属性

在讨论文件格式的细节之前，我们必须首先涵盖一个概念要点。在我们的系统中，每段文本可以拥有零到八个*属性*。属性，从最一般的角度来说，传达关于文本的额外语法信息。最初，这些属性源于正确处理复数形式的需要。在英语中，我们写 "1 apple" 和 "2 apples"，这意味着我们不能简单地将一个数字代入像 `___ apples` 这样的空白中，因为它有时会产生不合语法的结果。这个问题的解决方案不是写单个 "apples"，而是同时写 "apple" 和 "apples"，并根据填入空白处的文本的*属性*来选择使用哪个（在这种情况下，它是单数还是复数值。

有许多属性是基于形成基数和序数形式所需的类别[可以在这里找到](https://www.unicode.org/cldr/charts/44/supplemental/language_plural_rules.html)。（基数形式像 "apple" 与 "apples"，而序数形式是判断在数字后写 "st"、"nd"、"rd" 或 "th" 所需的信息。）

为方便起见，以下属性内置于数字中：`zero`、`one`、`two`、`few`、`many` 和 `other` 用于基数类别，`ord_zero`、`ord_one`、`ord_two`、`ord_few`、`ord_many` 和 `ord_other` 用于序数类别。大多数数字都会带有每一类中的一个。请注意，尽管它们有名称，但这些属性的含义完全取决于上一个链接中描述的规则，并且因语言而异。最后，还有 `z`（即真正的零），应用于表示零的文本（不要与前面提到的 `zero` 属性混淆，它做的工作不同）。提供 `z` 是为了让你可以生成像 "no apples" 这样的文本，而不是 "0 apples"，如果你愿意的话。

请注意，最多可以定义 255 个属性。你尝试使用的任何属性都会起作用，因为未知属性在遇到时会自动定义。你可以使用这些属性来代入具有正确语法性别的形容词来修饰它们所修饰的名词，使用正确的 "a" 或 "an" 冠词，等等。

### 键和内容

每个文件由多个*键*及其附带的文本*内容*组成。文件中的所有文本都必须是 utf8 编码。键通过一或多个空白字符与其内容分隔，内容通过一或多个换行符与下一个键分隔。键由键名组成，可选地后跟一或多个属性，每个属性之间以及属性与键名之间用 `.` 字符分隔。例如，`apple` 是一个仅由键名 `apple` 组成的键。`apple.many` 是键名 `apple` 加上属性 `many`。而 `apple.many.def` 是键名 apple 加上两个属性：`many` 和 `def`。键名可以在文件中多次出现。单个键名的所有键会组合在一起，可以通过指定要匹配的属性来选择与该键名关联的特定内容之一，如下所述。请注意，键名不能包含以下字符：`.`、`?`、`{`、`}` 或 `\`。

### 特殊内容序列

除非另有说明，键的内容将按原样在程序中重现。例外情况与字符 `\`、`{` 和 `}` 有关。一般来说，你可以将 `\` 字符视为启动一个特殊命令，使显示的文本做出你无法通过字面表达的事情。

#### 简单替换

有三个命令是 `\` 后跟单个字符：`\\`、\n` 和 `\t`。遇到这些命令时，它们会被替换为单个字符。`\\` 变成单个 `\` 字符，`\n` 变成换行符，`\t` 变成制表空格。请注意，如果这些命令后面紧跟着另一个命令，例如 `\n\t`，那么它们必须通过与后续文本分开的空格与文本分隔。例如：`Line 1\n Line 2`。`\n` 后面的空格不会出现在文本中；在上述示例中显示时，"Line 2" 前面不会有空格。

#### 格式命令

以下命令都写成后跟一对花括号（`{ ... }`），它们应用于花括号之间的文本。
- `\it{ ... }`：花括号内的文本如果可能将以斜体显示。
- `\b{ ... }`：花括号内的文本如果可能将以粗体显示。
- `\sc{ ... }` 花括号内的文本如果可能将以小型大写字母显示。
- `\os{ ... }` 花括号之间的文本中出现的数字如果可能将以旧式数字显示。
- `\tab{ ... }` 花括号之间的文本中出现的数字如果可能将以制表/齐线数字显示。
- `\sup{ ... }` 花括号内的文本如果可能将以上标显示。
- `\sub{ ... }` 花括号内的文本如果可能将以下标显示。

#### 特殊字符

`\c{ ... }` 命令用于插入可能难以输入的特殊字符。`...` 可以是以下之一：
- `left-brace`：插入 `{` 字符
- `right-brace`：插入 `}` 字符
- `em-space`：插入一个 em 空格
- `en-space`：插入一个 en 空格
- `3rd-em`：插入一个三分之一 em 空格
- `4th-em`：插入一个四分之一 em 空格
- `6th-em`：插入一个六分之一 em 空格
- `thin-space`：插入一个薄空格
- `hair-space`：插入一个发空格
- `figure-space`：插入一个数字空格（即如果字体支持，一个齐线数字的宽度）
- `ideo-space`：插入一个表意空格（适合 CJK 字形）
- `hyphen`：插入一个连字符
- `figure-dash`：插入一个数字破折号（即如果字体支持，一个齐线数字的宽度）
- `en-dash`：插入一个 en 破折号
- `em-dash`：插入一个 em 破折号
- `minus`：插入数学减号
- `####`（任何十进制数字）：插入该值的 unicode 码点

#### 其他格式

- `\o{ ### }{ ... }`：这是用于更改文本颜色的命令。第二对花括号内的文本将使用第一对花括号中指定的数字对应的调色板条目进行渲染。
- `i{ ### }`：此命令在文本中的位置插入由花括号之间的数字指定的图标。请注意，与给定数字对应的图标取决于文本出现的上下文（你不能依赖 `i{0}` 始终产生相同的图标）。如果将要出现的图标的性质对翻译很重要，你需要向其中一位程序员咨询。
- `\a{ name }`：此命令为文本附加额外属性（它不会对显示的文本产生可见效果，并且它出现在哪里并不重要，所以我建议将所有此类命令放在末尾）。这是从变量替换（下面讨论）中获取信息的一种方式。例如，如果我们的文件包含 `apple.many apples\a{vowel}`，那么我们可以针对提供的属性（`vowel`）进行匹配，以检测键名解析到的术语是否以元音开头。（请注意，我不预计在实践中这会经常用到。）

#### 变量替换

游戏中的许多文本不仅仅是通过直接获取键的内容来生成的，而是将其用作模板，将各种细节（如名称和数字）插入其中。为了使其工作，内容暴露出应该插入信息的位置，称为*变量*。因为可能需要在同一位置插入多条信息，每个变量都有一个唯一的名称。这些变量的名称由使用文本的代码确定。因此，变量本身的名称不得被翻译或更改。

从功能上讲，变量替换的位置简单地由 `{ ... }` 表示，其中 `...` 是变量的名称。例如 `{number} percent complete` 可以与变量 `number` 一起使用来创建类似 `5 percent complete` 的文本。你也可以通过在变量后面附加 `.` 和属性来添加一或多个属性。例如 `{value.a.b}` 是变量 `value` 的变量替换，带有额外的属性 `a` 和 `b`。如果插入变量的文本来自文件中其他地方的键（对于几乎所有非数字的内容都是如此），那么这些属性将用于从该键族中选择最匹配属性的那一个。如果文件中有 `apple apple` 和 `apple.many apples`，那么将键 `apple` 传递给 `{value.many}` 将导致 `apples` 出现而不是 `apple`。这也可以用来匹配修饰名词的形容词的语法性别，例如。

#### 替代选择

最后，我们来到本地化系统中最复杂的部分：根据属性选择替代文本序列。从抽象层面来说，这个机制的作用是在较长的内容中允许我们根据变量的属性，适当地生成例如 "apple" 或 "apples" 的文本。这在语法上是通过 `\match` 命令完成的。每个 `\match` 命令后面跟着任意数量的花括号对 `{ ... }{ ... }`。每对的第一个成员包含匹配的*条件*，第二个成员包含匹配的*内容*。当遇到 `\match` 命令时，会按顺序检查每对的条件，第一对条件满足的内容将被替换到文本中（忽略所有其他对的内容）。

##### 匹配条件

每对的条件是 0 个或多个属性测试。每个测试都表示为一个变量（就像你为变量替换所写的那样），后跟一个 `?` 字符，然后最多跟 8 个属性，属性之间用 `.` 分隔。对于条件的每个部分，变量将像变量替换正在发生一样被查找（因此 `name.attr?...` 可以用来选择键集的特定成员）。然后，将检查它是否拥有 `?` 之后列出的每个属性。如果有，那么它的部分条件就是匹配的。如果条件的所有部分都匹配，那么该条件的内容将被替换到文本中，并且 `\match` 命令的剩余匹配对将被跳过。如果条件的某个部分不匹配，那么过程将重复下一个匹配对。请注意，一对空花括号 `{}` 将始终匹配，因此可以用作一对中的最后一个条件，以确保 `\match` 命令的某些内容总是被使用。

##### 匹配内容

除进一步的 `\match` 命令和 `\a` 命令外，所有命令都可以出现在匹配内容中。

##### 示例

考虑内容 `I have \match{number?one}{ an apple }{number?z}{ no apples} }{}{ {number} apples }.`。假设变量 `number` 是数字 `0`、`1` 或 `3` 并具有适当的属性，该内容将为每种可能性产生：
```
I have an apple.
I have no apples.
I have 3 apples.
```
现在，让我们考虑一些更复杂的内容：内容 `I have \match {number?one}{ {obj.ind} } {number?z}{ no {obj.many} } {}{ {number} {obj.many} }.` 并且假设还定义了以下键和内容：
```
apple apple
apple.ind an apple
apple.many apples
lemon lemon
lemon.ind a lemon
lemon.many lemons
```
和之前一样，让我们考虑当变量 `number` 是数字 `0`、`1` 或 `3` 并具有适当属性，并且假设变量 `obj` 被分配给键 `apple` 或键 `lemon` 时会发生什么。在这些情况下，将产生以下文本：
```
I have an apple.
I have no apples.
I have 3 apples.
I have a lemon.
I have no lemons.
I have 3 lemons.
```
