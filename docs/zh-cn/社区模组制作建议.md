## 已识别的问题

一些问题已经被 Project Alice 解决，有些还没有。欢迎在下方添加你的惨痛经历，供 1.0 版本后考虑：

- 没有原生动态本地化支持（不使用 change_region_name 的情况下）
- 贸易窗口在商品数量过多时会溢出
- 未记录（Alice 已经列出了所有）且不一致的修正（有些只能在科技和发明中使用）
- 必须为每个文件夹添加 1 项科技，否则 UI 会损坏
- 选择让盟友加入哪些战争

目前正在解决的问题：
- 不支持 Unicode：随 UI 重做一起到来
- 生产商品标签页仅支持 N 种商品：见上文

# Project Alice 社区模组制作建议

**给每位模组制作者的通用提示，供参考**
一直在向模组制作者询问他们对 Victoria 2 可修改性的建议，因为还有另一个 OpenV2 尝试，我不知道它是否会成功，但我想在力所能及的范围内帮助他们。
Ctrl+c ctrl+v 部分：你或其他你认识的模组制作者，对 Victoria 2 的可修改性有什么建议？
生硬的开场白到此为止，我想说明一点：整体游戏机制将尽可能保持接近原版 Victoria 2。他们计划让 Victoria 2 的模组与 OpenV2 兼容。最后注意，该项目尚处于早期阶段，这是为了规划目的。
本文档将涵盖模组制作的各个方面，但你也可以提出非模组化的 QoL 和多人游戏方面的建议。
脚本编写 — 这包括事件、决议等所有内容，也包括 on-actions 和 UI 事件，IF 运算符、<>、更多作用域、条件、修改器、效果等。
经济 — 任何与商品、建筑、生产、贸易、金钱以及人口与经济关系有关的内容。
政治 — 任何与政治、运动/叛乱界面、决议、释放国家窗口以及人口政治或类似新增内容有关的内容。
人口 — 任何与人口发展方向及其模组制作有关的内容，例如迁移、晋升、工作、政治倾向，它们与其他人口的关系，是否被接受以及谁在经济上拥有什么等。
科技 — 只是科技，你希望如何修改研究点和文明机制。
军事 — 军事内容，例如拥有修改掷骰数量的定义或更好的将领生成。
地图 — 地形能力、省份的制作或定义方式、河流、州等。
外交 — 你可以修改的行动、和平协议、战争借口、恶名、危机等。
图形与 GFX — 任何你能想到的外观和 GFX 相关内容。
UI — 他们已经在计划 UI 了，但你可以提出自己的想法。
杂项 — 任何其他模组制作或建议方面的内容。

## Rascalnag：

图形与 GFX — 更可定制的着色器管线。说实话，Victoria 3 在这方面做得很好，因为你可以相当容易地替换和添加新的着色器文件。所以类似的灵活性会很好。还能更容易地/能够添加新纹理（不仅仅是替换现有纹理）。

UI — Victoria 2 的 UI 模组制作对我来说通常很糟糕，因此应该对制作 UI 元素的过程进行大改。

军事 — 这应该相当模块化。例如，能够添加一个补给线模块，根据是否存在能够供应单位需求的实际补给线来按比例调整单位在某省份的损耗/战术，这会很棒，并鼓励随着科技的发展向可持续的前线战略过渡，因为单位会变得更"饥饿"。但这需要一个既能代表 Victoria II 基础战争又能代表如此大偏离 Victoria II 方法的系统，这是一项复杂的任务。特别是考虑到 AI 需要配套的模块化才能与任何此类插件配合工作。

## Kiwi/ColdSlav

### 模组功能 — 经济
- 能够检查商品价格，无论是固定金额还是由商品定义默认价格的确定百分比。
- 这也意味着能够限定商品，无论是单独限定，还是按组（或模组制作者认为合适的其他因素）限定。
- 能够人为控制商品价格（肯定对经济系统无害），方式同上。
- 能够限定工厂，并按类型检查它们：生产的商品、消耗的投入商品（按模组制作者定义的时间量）、产出的商品（与投入方式相同）、盈利能力、雇员、雇员比例、是否存在某类雇员、现金储备、是否可升级、是否在建、建造时间？、（在国家中的）数量，等等？
- 不仅能够在首都建造工厂，还能在任何州建造。
- 能够修改每州的工厂数量，或者更好（如果容易实现的话），能够在每个州的基础上设置工厂数量上限。
- 能够检查在州层面、国家层面和国际层面产量最高的商品（这在原版 Victoria 2 中已经某种程度上存在，但几乎未被使用，且其工作方式 poorly understood）。
- 可修改的工资，能够定义基本工资值，并能够在人口类型、国家、工厂以及可能的地区基础上修改它们。
- 修复收入修正器，并包含更多同类型的修正器，但按人口类型而非仅按阶层划分。
- 更多经济模组制作。
