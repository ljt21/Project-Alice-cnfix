## 超级控制台

### 玩家指南

在 1.1.3 版本中，游戏内控制台（按下 `~` 时弹出的窗口）已被完全重做。控制台不再支持固定的命令列表，而是支持一种完整的编程语言，可以让你检查和修改游戏的几乎任何部分。对于普通作弊者来说这可能有点多，因此本指南的这一部分将指导你如何让新控制台完成与旧控制台相同的事情。请注意，新控制台对大小写、空格和值的精确格式很敏感（10.0 给你浮点值，而 10 给你整数值，期望其中一种的命令不会接受另一种）。如果你的控制台输入有效，你会看到一个绿色的 `ok.`。否则，你会收到一个错误。如果你遇到错误，请仔细检查你的拼写和间距，然后重试。

#### 更改恶名、威望和其他数值

以下是如何使用控制台更改一些常见数值：

- `10.0 TAG plurality !` : 将三个字母标签的多元性设置为 10.0（注意，需要小数位）
- `10.0 TAG prestige !` : 将三个字母标签的威望设置为 10.0（注意，由于永久威望修正，显示的威望可能不会完全反映这一点）
- `10.0 TAG infamy !` : 将三个字母标签的恶名设置为 10.0
- `10.0 TAG 0 >commodity_id stockpiles !` : 将三个字母标签的国库设置为 10.0
- `10.0 TAG research_points !` : 将三个字母标签的研究点数设置为 10.0
- `10.0 TAG diplomatic_points !` : 将三个字母标签的外交点数（你为外交行动支付的点数）设置为 10.0
- `10.0 TAG leadership_points !` : 将三个字母标签的领导力点数（你为将军和海军上将支付的点数）设置为 10.0

在控制台中显示所有商品 ID 的命令：
`commodity-size while 1 - dup 0 >= loop dup dup >commodity_id @ name @ localize swap end-while drop`

在上面看到 `TAG` 的任何地方，你也可以用 `player @` 代替（注意空格），它将作为你当前扮演的任何国家的标签。在上面看到 `PROV123` 的任何地方，你也可以用 `selected-prov @` 代替，它将作为主用户界面中当前选中的省份。另外，关于省份名称的快速说明。能够引用 `PROV123` 依赖于它在控制台首次打开时是该省份的名称。最初，所有省份都按照该模式命名（PROV 加上它们在游戏文件中的 ID）。然而，由于重命名效果，这可能会改变。如果你加载了一个省份已被赋予新名称的游戏，你将不再能够用其原始的 `PROV123` 名称引用它。
 

### 超级用户和开发者指南

新控制台嵌入了一种完整的编程语言。我不打算在这里完整记录该语言。相反，本节将提供你需要了解的概述，以访问存储在内部数据容器中的所有内容并编写你自己的简单函数。对于开发者来说，这可能不足以替代调试器的正确使用。然而，它可以使你的一些调试任务变得更容易，例如查找与给定 id 关联的特定属性值，或确定特定键对应的文本是什么。

#### 基础：栈语言

控制台中使用的语言是一种静态类型的面向栈的语言。该语言所做的几乎所有事情都涉及对*数据栈*的操纵。我们将在本文档中将数据栈的内容描述为一系列水平排列的方框。最左边的方框是栈底，而最右边的方框是栈顶。项目总是从栈顶添加和移除。

```
┌─────┬─────┬─────┐
│  A  │  B  │  C  │
└─────┴─────┴─────┘
```

例如，上面是栈上有三个项目 A、B 和 C 的图示，其中 A 在栈底，C 在栈顶。

控制台中的一行被划分为所谓的*词*（words）。词之间由空白字符分隔。除此之外，组成词的字符没有任何限制（因此重要的是要记住，与 `C` 家族的语言不同，`a+b` 是一个单一的词，而不是三个词）。这些词也是区分大小写的，所以 `PROV123` 是一个与 `prov123` 不同的词。行中的每个词从左到右依次执行。这就是语言核心语法的全部内容。数字（整数和小数值）将它们的值压入栈。整数将 `i32` 类型的值（有符号 32 位整数）压入栈，小数将 `f32` 值（32 位浮点值，等效于 `C` 中的 float）压入栈。因此，行 `-3 4.5 100` 将产生以下栈：

```
┌───────────┬────────────┬────────────┐
│  i32: -3  │  f32: 4.5  │  i32: 100  │
└───────────┴────────────┴────────────┘
```

如果你在控制台中输入该行并按回车，你会看到每个值打印在下面，后面跟着 `ok.`。这是因为控制台中的一行执行后，栈上剩余的所有内容将被打印回控制台，后面跟着 `ok.` 以表示它已无任何错误地执行。如果你的命令在栈上没有留下任何内容，你将只看到 `ok.`。

#### 基础：数据类型

如上所述，该语言是静态类型的，尽管类型推断处理了大部分事情。然而，你需要知道该语言不做隐式类型转换，所以如果你想将不同类型的值相加，例如，你需要显式地将它们转换为相同类型。（注意，命令不会以不匹配的类型运行，所以如果你丢失了类型跟踪，你不会得到意外行为。）你可能遇到的常见数值类型有 `i8`、`i16`、`i32`、`i64`（各种位宽的有符号整数）、`u8`、`u16`、`u32`、`u64`（各种位宽的无符号整数）、`f32` 和 `f64`（分别对应于 `C` 中的 `float` 和 `double` 的浮点类型）。数值类型都可以相互转换，转换通常遵循 `C` 规则。你可能看到的其他类型有 `bool`（建模为无符号 1 位整数）和 `ptr(...)`，它是指向 `...` 类型的指针。例如 `ptr(i32)` 是指向 32 位整数的指针。`ptr(nil)` 等效于 `C` 中的 `void*`。这些是最常见的类型，但它们不是语言中的所有类型，用户还可以添加新类型。我们的数据容器接口自动为各种 id 和其他索引生成类型，我们稍后会回到这一点。

将值从一种数值类型更改为另一种数值类型是通过一系列 `>...` 函数完成的，其中 `...` 是目标类型的名称。例如 `>i32` 将值转换为 32 位有符号整数。在更传统的编程语言中，你会说这些函数中的每一个都有一个参数，即要转换的值，并返回一个值，即结果。然而，我们有一种面向栈的语言。因此，我们不再将函数视为具有单个参数，而是将其建模为从栈顶消耗单个值。并且我们不再将其视为返回单个值，而是将其视为向栈顶添加单个值。因此，我们将 `-3 4.5 >i32` 的执行建模为执行以下操作：

`-3 4.5` 将两个值压入栈
```
┌───────────┬────────────┐
│  i32: -3  │  f32: 4.5  │
└───────────┴────────────┘
```
然后，`>i32` 从栈中消耗一个值
```
┌───────────┐
│  i32: -3  │
└───────────┘
```
最后，`>i32` 将其结果推回栈
```
┌───────────┬──────────┐
│  i32: -3  │  i32: 4  │
└───────────┴──────────┘
```

#### 基础：数学运算

该语言支持基本数学运算 `+`、`-`、`*`、`/`、`mod` 和比较 `=`、`>`、`<=`、`<`、`>`、`>=`（注意，与 `C` 不同，相等比较是 `=` 而不是 `==`，不等是 `>` 而不是 `!=`）在相同数值类型的值对之间。像这样的二元运算消耗它们的两个栈输入并将结果放回栈上。对于前五个运算，结果将是相同类型，对于比较将是 `bool` 类型。栈顶的值是通常被认为是运算符"右手"的值，它下面的值是"左手"的值。（你可以将运算符想象为放在栈图示中最后两个项目之间，如果这有帮助的话。）

例如 `-3 4 -` 执行以下操作：

`-3 4` 将两个值压入栈
```
┌───────────┬──────────┐
│  i32: -3  │  i32: 4  │
└───────────┴──────────┘
```
`-` 移除两个项目以计算 -3 - 4。
```
╷
│
╵
```
`-` 将结果放回栈上
```
┌───────────┐
│  i32: -7  │
└───────────┘
```

#### 基础：栈操作

有时你会在栈上得到一些顺序不太正确的值。首选的解决方案是重写表达式，使值以它们需要被消耗的顺序留在栈上。当这不可能时，那么我们有栈操作词。最基本的操作词是 `dup`、`drop` 和 `swap`。

`dup` 复制栈顶的项目并将其推入栈，例如将
```
┌───────────┬────────────┐
│  i32: -3  │  f32: 4.5  │
└───────────┴────────────┘
```
变为 
```
┌───────────┬────────────┬────────────┐
│  i32: -3  │  f32: 4.5  │  f32: 4.5  │
└───────────┴────────────┴────────────┘
```

`drop` 与 `dup` 相反：它移除栈顶的项目而不对其做任何操作，将
```
┌───────────┬────────────┐
│  i32: -3  │  f32: 4.5  │
└───────────┴────────────┘
```
变为
```
┌───────────┐
│  i32: -3  │
└───────────┘
```

最后，`swap` 交换栈顶的两个项目，将
```
┌───────────┬────────────┐
│  i32: -3  │  f32: 4.5  │
└───────────┴────────────┘
```
变为 
```
┌────────────┬───────────┐
│  f32: 4.5  │  i32: -3  │
└────────────┴───────────┘
```

有时你需要更深入地挖掘栈。为此，我将简要介绍 `>r`、`r>` 和 `r@`，而不对它们说太多。除了数据栈之外，该语言还包含第二个栈，称为*返回栈*。这在传统上主要用于控制流，很少有词直接与之交互。`>r` 从数据栈移除顶部项目并将其推入返回栈顶部。`r>` 则相反：它从返回栈顶部移除项目并将其推入数据栈顶部。最后 `r@` 复制返回栈顶部（不更改它）并将副本推入数据栈顶部。

你应该始终努力将放在返回栈上的所有东西都取下来。留在返回栈上的项目不会显示在控制台中（尽管它们会被安全丢弃，所以不会干扰任何后续工作）。但是，暂时将项目放在那里是完全安全的。例如，你可能会遇到如下序列：`>r >r ...some work done here... r> r>`。这从栈顶取出两个项目并将它们临时存储在返回栈上，以便工作可以像它是两个栈项目更低一样执行。然后，当工作完成后，顶部两个项目被恢复到它们原来的位置（和顺序）。


#### 基础：指针

在我们开始检查和操纵游戏状态之前，需要了解的最后一个语言基础是指针。通常（虽然不总是），操作游戏状态的函数将立即返回的值将是指针，以便能够通过单个函数实现读取值和设置值。读取指针内容是通过词 `@` 完成的。`@` 从数据栈顶部获取指针值，解引用它，然后将指针的内容推入栈。你也可以将 `@` 视为用指针指向的值替换指针。词 `!` 用于将值存储到指针指向的位置。`!` 从栈顶消耗一个指针，并在其下面消耗一个相应类型的值。在使用 `!` 之前，栈图应如下所示：
```
┌┄┄┄┄┄┬─────────────────┬─────────────────┐
│ ... │     T: value    │ ptr(T): pointer │
└┄┄┄┄┄┴─────────────────┴─────────────────┘
```
这可能意味着你最终要写一行如 `value ...expression generating pointer... !`，其中 `!` 的使用与它最终将存储的值有一定距离。

如果你出于某种原因需要更改指针的类型，可以使用强大的 `ptr-cast` 词。`ptr-cast` 词后面必须跟一个指针类型的名称（例如：`ptr-cast ptr(i32)`）。指针转换函数然后取栈顶的指针并将其更改为指定类型。请注意，写入转换为另一种类型的指针很容易导致数据损坏。更改你要存储的数据类型以匹配指针类型几乎总是比反过来更好。

#### 与数据容器交互：id

类型会自动为每个数据容器键生成，名称相同（虽然不需要命名空间限定）。例如 `dcon::text_key` 在控制台中可作为类型 `text_key` 使用。任何此类键都可以直接使用 `>` 后跟键的名称作为单个词从 i32 值生成。例如 `0 >text_key` 将导致栈中包含一个用于第 0 个索引键的 `text_key` 类型。词 `>index` 执行逆操作，从栈中移除 id 并用其索引值替换它。

除了使用索引生成所需的 id 之外，你还可以使用它们的个体名称将某些 id 推入栈。具体来说，对于数据容器中大多数具有 `name` 属性的对象，你可以使用存储在那里的文本键（未本地化的键，不是其显示值）来获取该特定对象的 id。例如 `french` 将 French 文化的 `culture_id` 推入栈。三个字母的 TAG 标识符也基本上以这种方式工作，但代替推入 national_identity_id（这可能看起来是合乎逻辑的选择），它们推入当前持有该标签的国家的 `nation_id`，因为这更方便普通用户。

***重要说明：*** 开发者应记住，C++ 数据容器 id 内部存储的值*不是*其索引。相反，内部值 0 是无效 id，内部值 1 是索引零，依此类推。所以，如果你在调试器中看到一个 id 存储的值为 15，那将是命令 `14 >id_type` 生成的键。

词 `valid?` 从栈中取一个 id 值并推入一个布尔值，如果 id 包含有效索引则为 true，如果它是无效键的表示则为 false。如果你出于某种原因需要从控制台内部获取无效键，序列 `make key_name` 将其实例推入栈顶。

对于大多数 id，如果你将它们留在栈上，你只会看到单词 `#struct` 出现在控制台中，没有进一步的细节。然而，`text_key` 类型有一个例外。如果你将那种类型留在栈上，控制台将显示相应键值的未本地化内容。你还可以在 `text_key` 值上使用 `localize` 这个词。这将从栈中移除 id 并用一个代表键的本地化内容的新值替换它。如果该值留在栈上，控制台随后将显示该本地化内容。

#### 与数据容器交互：访问值

数据容器对象的所有属性都可以作为控制台中的词使用（非开发者：这些对象及其属性的描述中，应该有一份位于仓库 `src/gamestate/dcon_generated.txt` 文件中的描述，大多对非程序员也是可以理解的）。每个这样的词都会从栈顶消耗相应的 id 类型，并推入一个指向该对象实例该属性值的指针。例如，`0 >national_identity_id adjective` 将返回一个指向第 0 个国家身份对象的默认形容词 `text_key` 存储位置的指针。如果你不打算修改该值，这几乎总是后跟一个解引用操作，所以你的控制台命令可能看起来像 `0 >national_identity_id adjective @` 甚至 `0 >national_identity_id adjective @ localize`。一般来说，属性的数据类型与前面章节中描述的数据类型的对应方式应该是显而易见的。

对于大多数属性来说，这就是你真正需要知道的一切，但有一些棘手的情况。首先，有数组类型属性，例如
```
property{
	name{ government_adjective }
	type{ array{government_type_id}{text_key} }
}
```
对于这样的属性，你既需要对象实例的 id，也需要数组的索引（在这种情况下是 `government_type_id`）。数组的索引应该是栈顶的项目，对象实例的 id 紧接在其下方。对于我们这里的例子，我们可能会写类似 `0 >national_identity_id 1> government_type_id government_adjective @` 的东西。

从池化向量类型属性检索信息也有额外的复杂性。对于诸如（来自 leader_images 对象的）属性
```
property{
	name{ generals }
	type{vector_pool{2000}{gfx_object_id}}
}
```
`generals` 这个词将产生一个代理类型，代表为特定对象实例存储的单个数组。然后 `size` 这个词将弹出该代理并在其位置放入一个 `i32`，包含该特定数组的大小。或者，如果在代理对象之上向栈推入一个 `i32` 值，`index` 这个词将返回指向数组该索引处存储值的指针。例如 `0 >leader_images generals size` 将产生第 0 个 leader images 对象实例存储的将军图像数量。`0 >leader_images generals 2 index @` 将返回为第 0 个 leader images 对象实例存储的第 3 个将军图像的值（一个 `gfx_object_id`）。

与对象的属性一样，可以遍历对象参与的关系。不幸的是，事情可能稍微不那么直观。C++ 接口自动生成许多在控制台接口中缺失的辅助函数，使事情稍微冗长一些。

要从对象实例转到它参与的关系，会生成一个词，其名称由关系名称 + `-` + 关系中链接的名称组成。例如，`culture_group_membership-member` 是将 `culture_id` 转换为 `culture_group_membership_id` 的词。然而，这个词产生什么取决于链接的性质。普通的单项目链接将产生指向 id 的指针。但如果该链接也是关系的主键，那么它将只产生 id 本身作为值，而不是作为指针（因为链接到的 id 是计算出来的，不是存储的，所以没有内存可指向）。最后，如果这是一个多对一链接，那么这个词将产生一个数组代理，如上所述的 vector pool 类型属性。这就是如果你使用 `culture_group_membership-group` 这个词在一个 `culture_group_id` 上会得到的结果，即一个数组代理，存储涉及该组的所有 `culture_group_membership_id`。

坦率地说，可能需要一些实验才能熟悉如何从特定关系中提取数据。作为指南，你还可以查看 `fif_dcon_generated.hpp` 文件的内容，该文件包含使所有访问属性的词生效的定义。虽然你可能不完全理解这些定义中到底发生了什么，但我确实认为，凭借本文档中包含的信息，你可以弄清楚一个词是执行索引计算以返回值，还是生成数组代理对象，还是返回指向特定值的指针。

将关系的 id 转换为参与其中的对象实例的 id 则不那么冗长：链接的名称本身就充当执行此操作的词。

#### 与数据容器交互：更复杂的函数

正如上一节所示，遍历关系以从一个 id 移动到另一个 id 可能变得非常冗长，而且肯定不是你想在控制台中反复输入的东西。既然我已经为控制台启用了 ctrl+v，一种解决方案是创建一个包含你创建的命令的文档。然而，你也可以在控制台中声明新函数，将你编写的命令打包成单个词（尽管请注意，这不会在一次会话之外持久存在）。这样做的语法是：
```
: word-name .... contents .... ;
```
`:` 后面的词将是将来调用你定义的词。`;` 之前的剩余词是每当你的新词被使用时将执行的命令。让我们通过一个示例来演示如何使用它。假设，承接上一节，你希望能够列出属于特定文化组的各个文化的名称。要为其中一个名称执行此操作，你可能输入 `0 >culture_group_id culture_group_membership-group 2 index @ member name @ localize`（这将获取第三个文化的本地化名称）。显然，这样做是相当大的打字练习，你不会想反复输入它。所以，首先我们需要提取出"变量"。假设你希望能够更改 `culture_group_id` 和你正在查找的文化的索引。并且假设你还想以 `0 >culture_group_id 2 member-name` 的方式使用你的词。在这种情况下，索引将在栈顶，id 将在你的词运行时位于其下方。我们首先从命令中切掉硬编码的文化组 id，生成 `culture_group_membership-group 2 index @ member name @ localize`。那里的第一个词期望组 id 在栈顶，而不是索引。所以我们将首先通过 `>r` 将索引移开，得到 `>r culture_group_membership-group 2 index @ member name @ localize`。当我们执行这些命令时，最终会到达 `2`。但我们不希望它总是 2，我们想要之前放在返回栈上的索引，所以我们将 2 替换为 `r>`。这给了我们 `>r culture_group_membership-group r> index @ member name @ localize`。我们像这样将它变成一个定义：`: member-name >r culture_group_membership-group r> index @ member name @ localize ;`。一旦我们按回车，控制台告诉我们 `ok.`，然后我们就可以使用我们的新词了。请注意，词在首次使用时"编译"，而不是在用 `:` 定义时编译，所以任何错误将在你首次尝试使用时才会暴露。我建议在将命令转换为词之前先测试它们以首先发现任何错误。
