## 关于 Project Alice

### 入门指南

请参阅 docs/contributing.md 获取更多信息！该文件包含在 Windows 和 Linux 上编译项目的详细信息。

### 延续 Open V2 的工作

Project Alice 的目标（以 [Alice Roosevelt Longworth](https://en.wikipedia.org/wiki/Alice_Roosevelt_Longworth) 命名）本质上是创建 [open v2](https://github.com/schombert/Open-V2) 的新版本——这是我之前创建 Victoria 2 克隆版的项目——然后完成一个可运行的、功能完整的游戏版本。这意味着，至少在一开始，我们将很少偏离 Victoria 2 的做法，只是为了保持项目的专注和正轨。一旦完成 1.0 版本，我们就可以用它作为新实验的试验场。截至 2023 年 7 月，我基本上已经追上了 open v2 的进度，除了用户界面方面的一些不足。

我希望找到一些人组成一个美术团队，能够以新的艺术风格重新制作 Victoria 2 使用的所有现有 UI 资产，为这个项目赋予自己独特的视觉标识。我知道这不是一件小事，但另一方面，不需要任何 3D 建模工作。我们已经可以从 SOE 项目中获取一套新的旗帜资产来使用。

### 与其他项目的比较

与 SOE（Symphony of Empires）相比，这个项目更像是一个直接的 Victoria 2 克隆版，而 SOE 是它自己的游戏。不过，我倾向于把它视为一个姐妹项目。我们这里有些人同时参与了两个项目（我本人也会偶尔提出建议），如果你看到我们做了什么对他们有用的东西，我相信他们会很感激你的贡献。同样，我们至少也会使用他们的一些作品。

至于 OpenVic2 项目，虽然它被"营销"为 Victoria 2 的"克隆版"，但这并不是他们正在努力创建的东西。相反，他们打算提供一整套全新的资产，包括事件、决策等，这将使他们的项目更像一个 Victoria 2 风格的游戏。它的管理方式也截然不同。如果你对 Project Alice 和 OpenVic2 都感兴趣，我建议你花时间在两个团队中，甚至可以同时加入两个项目。

### 许可证

作为本项目的一部分产生的所有代码将根据 GPL3 许可证发布（与 open v2 相同）。美术资产的许可证将由美术团队决定，但我希望采用某种形式的 Creative Commons 许可证。

### 编程工具

项目本身使用 CMake（是的，我知道我们都讨厌 CMake）。理想情况下，它应该能够使用三大编译器中的任何一个进行编译：MSVC、Clang/LLVM 或 GCC。我个人倾向于使用 MSVC，但我也使用 Windows 上的 Clang。确保 GCC 上的一切正常工作将是别人的问题，如果没有贡献者最终成为常规 GCC 用户，那么 GCC 兼容性可能会从列表中消失。

就我个人而言，我使用 Visual Studio 2022（免费的社区版）来处理这个项目，但如果你正在贡献，你应该能够使用你喜欢的任何工具，包括 VS Code、CLion、EMACS、VI 等。

### Project Alice 会在哪些方面偏离 Victoria 2？

最初，Project Alice 将尽可能地模仿 Victoria 2 的机制，只有一些小的例外。是的，我们可以改进的地方有很多。我在 open v2 中的心态是，在开发过程中这里改一点那里改一点，这是项目偏离轨道并最终被放弃的一个主要原因。我不想再犯同样的错误，所以这次我会尽我所能抵制在我们拥有一个可运行的、功能完整的游戏之前进行改进的冲动。话虽如此，有一些改变是我们无论如何都必须做的。全球地图太有趣了，不加入不行。还有一些生活质量改进（比如可以点击跳转到事件所谈论内容的超链接，或者事件触发原因的摘要）足够简单也足够有用，不加入真的很可惜。但这一次不会有重大机制调整！我保证！
