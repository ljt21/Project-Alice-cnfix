# 下载

要下载 Project Alice 的预编译版本，请在 [Project Alice 仓库](https://github.com/schombert/Project-Alice) 中找到"标签"按钮。

![image](https://github.com/schombert/Project-Alice/assets/39974089/0d393492-0471-44b2-b222-4efe6ed57c92)

选择你想下载的特定版本，我们建议获取最新版本（最上面的一个），以获得更愉快的体验：

![image](https://github.com/schombert/Project-Alice/assets/39974089/4f179bf2-773b-4b2c-84f1-142cde855918)

点击"下载"文本后，你会发现可能有 3 个文件，其中两个是 ZIP 和 TAR 格式的源代码文件，就我们的目的而言，这些可能对你没用——如果你需要，应该查看构建指南。所以，下载那个**不是**源代码的文件。

# 安装

下载完成后，解压它，你会得到以下文件（README 有时不包含在内）。

![image](https://github.com/schombert/Project-Alice/assets/39974089/d669abdd-647a-4417-ac8a-2a5c9c6e6fec)

将所有这些文件拖放到你的 Victoria 2 目录中，然后它应该看起来像这样

![image](https://github.com/schombert/Project-Alice/assets/39974089/3dd79391-999b-410e-8bd1-020437d34b89)

然后只需点击 `launch_alice` 可执行文件即可。

- `Alice`：主游戏可执行文件，不要直接打开。
- `launch_alice`：游戏启动器，你需要为每次修改模组后创建/重新创建场景。
- `dbg_alice`：一个小型调试工具，用于帮助追踪错误，可选，没有它也没关系。
- `readme OR ELSE.txt`：包含相关信息，同样是可选的，没有它也没关系。
- `assets`：Alice 正常工作所**必需**的文件夹。

### （高级）Windows 启动器命令行选项

- `-modsMask {mask}`：启用所有匹配该掩码的模组。
- `-outputScenario {name.bin}`：以定义的名称输出场景。
- `-autoBuild`：如果设置，自动构建场景并关闭启动器

### （高级）Windows 构建命令行选项

- 指定 .bin 场景的名称以启动它。
- `-host`：设置后，作为主机启动多人游戏
- `-join {ip}`：加入多人游戏
- `-name`：设置玩家名称
- `-password`：输入大厅密码
- `-player_password`：设置玩家密码以保护你的国家
- `-v6`：使用 IPv6
- `-v4`：使用 IPv4
- `-headless`：不渲染游戏 UI，启动后立即开始游戏
- `-repeat`：如果是 headless 模式，在上一个游戏结束后立即重新开始
- `-speed`：设置游戏速度

## Linux

### 使用 Proton
- 按照与 Windows 相同的步骤进行安装和下载
- 在 Steam 上点击添加游戏 --> 非 Steam 游戏，然后选择 'launch_alice.exe'
- 右键点击 --> 属性 --> 兼容性 --> 强制使用 Steam Play 兼容性工具
- 选择 Proton

#### 不使用 Proton (Distrobox)
- [请按照这里的说明操作](https://github.com/ComradeNiobe/project-alice-distrobox)

### Ubuntu/PopOS

要在 Ubuntu/PopOS 上安装游戏，请按照与 Windows 相同的步骤进行安装和下载，直到启动 `launch_alice` 这一步。此时，下载 [Lutris](https://lutris.net)。

启动 Lutris，点击侧边栏中的 Wine 图标。一个向下箭头图标会出现，点击它。你应该已经安装了一个版本，但如果没有，请选择一个。据说 ge 版本是许多游戏的绝佳基础。

现在点击主菜单左上角的 + "手动添加游戏"图标。然后，选择最后一个选项"添加本地安装的游戏"。在第一个窗口"游戏信息"中为游戏命名（可以是任意名称）。从下拉菜单中选择"Wine"作为运行器。在第二个标签"游戏选项"中找到你的 `launch_alice.exe` 文件，并将其选为"可执行文件"。忽略参数和工作目录。"Wine Prefix"我选择例如 ~/Games/Project-Alice，并选择 64 位作为前缀架构。在"运行器选项"下，如果你有不同版本的 Wine，可以选择不同的版本。忽略"系统选项"，如果你需要，可以稍后进入。"保存"，然后它会退出。如果你在 Lutris 的列表中看不到你的游戏，请重启应用。有一个小 bug，在你至少"安装一个游戏"之前不会显示启动横幅。目前尚不清楚这个 bug 是否仍然存在。

现在，游戏应该可以从 Lutris 主菜单中运行了。

## 操作控制

- 拖动鼠标：这会显示一个白色轮廓的拖动框，选择单位，优先选择陆军而非海军
- 拖动鼠标时按住 Ctrl：优先选择海军而非陆军
- HOME：前往你的首都
- Ctrl-0, Ctrl-1, ..., Ctrl-9：将单位添加到控制组 #1、#2、...、#9。将显示在大纲中单位旁的括号内，一个单位可以属于多个组
- 0, 1, ..., 9：选择控制组 #1、#2、...、#9 中的单位
- Shift-0, Shift-1, ..., Shift-9：追加选择控制组 #1、#2、...、#9 中的单位
- 方向键：在地图上移动
- WASD：移动地图（只有在设置中启用了 WASD 移动时才有效）
- 按住 Shift + 方向键：在地图上移动得更快
- 按住 Shift + WASD：同上，但只有在启用了 WASD 时才有效
- PgDown/PgUp：放大和缩小
- Pause/Spacebar：暂停或取消暂停游戏
- Escape：打开主菜单
- F1：打开生产标签页
- F2：打开预算标签页
- F3：打开科技标签页
- F4：打开政治标签页
- F5：打开人口标签页
- F6：打开贸易标签页
- F7：打开外交标签页
- F8：打开军事标签页
- F9：打开总账
- F10：打开宏建造器
- 小键盘 +：增加速度
- 小键盘 -：减少速度
- s：拆分单位
- r：重组单位（从现有单位创建新单位）
- Tilde (~)：打开控制台
- TAB：打开大厅。

其他控制直接显示在工具提示上（例如，右键点击将科技添加到队列）。

## 多人游戏说明

### 加入游戏

启动启动器后，你会看到类似以下的内容：

与单人游戏一样，你首先需要为你想玩的模组组合创建一个场景（如果你还没有创建，"开始游戏"、"主机"和"加入"按钮将被禁用）。这个模组组合**必须**与你尝试加入的游戏匹配（包括模组的版本；记住，修改模组文件是**不够的**——你还需要重新构建场景）。

然后，更改你的用户名（目前，文本框有点奇怪：你需要将鼠标悬停在它们上面然后输入——有一个看不见的光标，表现得就像它总是在文本末尾），并输入你想连接的主机的 IP 地址（如果你从其他地方复制了 IP 地址，ctrl+v 会将其粘贴到地址框中）。

加入游戏会将你带入国家选择界面，不同之处在于你会看到其他玩家和主机。只要某个国家没有被其他玩家控制，你就可以选择任何你想玩的国家，因为两个玩家不能控制同一个国家（目前）。

作为客户端，你将无法控制游戏速度，也不能自行加载存档文件——主机对游戏会话拥有绝对权力。主机可以随时踢出和封禁玩家，他们也可以暂时返回大厅，允许玩家选择其他国家/切换。

多人游戏中始终开启战争迷雾。

注意：如果你输入 IPv6，你的主机也需要接受 IPv6 连接，否则会失败。IPv4 也是如此，通常出于用户体验考虑推荐使用 IPv4。
注意：不支持域名，请手动解析并输入其对应的 IPv4（这是未来的功能）。

#### 主持游戏

如果你使用 IPv4，在 IP 字段中输入一个点 "."，如果你使用 IPv6，在 IP 字段中输入一个分号 ":"。使用 IPv4 的客户端无法连接到 IPv6 主机。

要接受客户端的连接（不使用 Hamachi 等隧道服务），你需要能够在本地应用文件夹中 host_settings.json 指定的端口上接受连接。默认端口是 1984。通常这需要在你的路由器中将该端口转发到你的本地机器。你可以使用[开放端口检查工具](https://www.yougetsignal.com/tools/open-ports/)来检查你是否正确打开了端口，请注意这类工具可能不是 100% 可靠的，因此让另一个人帮助测试是非常推荐的。

在给玩家你的 IP 地址以便他们连接到你时，你需要提供你的机器从互联网角度看的地址（即不是你的本地子网地址）。

作为主机，你将能够在游戏中和大厅中踢出和封禁玩家。确保事先准备好一切。

请注意，加载存档文件会将存档文件发送给每个你已连接和将连接的客户端，例如，如果你有 8 个玩家，你将发送 8 次存档文件，通常约为 3 MB，因此请注意，每次加载存档文件时，你可能最终发送 3*8=24 MB 的信息。它已经使用可用的最大压缩级别进行压缩，因此没有必要进一步压缩（根据压缩的数学定律，它有自己的维基百科文章，我就不赘述了）。

对于处理捣乱者：
- 踢出：将玩家从会话中断开，如果他们愿意，可以重新加入。
- 封禁：将玩家断开连接并将其列入黑名单，除非开始新的会话，否则他们将无法重新加入。基于 IP（或如果使用 IPv6，则基于 MAC）——再次说明，任何足够坚定的人都能够绕过这一点，但它适用于大多数实际情况。

#### 故障排除

对于客户端：
- 检查你输入的 IP 是否正确
- 检查你的玩家名称不是 a) 空的，或 b) 不合适的
- 检查你未被服务器封禁
- 确保你使用的是 IPv4 或 IPv6——相应地主机不会接受使用不同 IPv 的客户端
- 如果使用 IPv6，检查你的路由器是否支持它
- 如果使用 IPv4，不要输入分号（例如 `127.0.0.1:1283`），因为这会被解释为 IPv6 地址
- 如果主机使用自定义端口，则用分号指定端口号。例如 (`127.0.0.1;1983`)。如果未指定端口，则默认为 1984。
- 检查你的防火墙没有阻止 Alice
- 检查你的互联网连接（甚至可以 ping 主机）
- 如果使用 VPN 隧道应用，检查你是否已连接到对等方
- 确保你们都有相同的游戏版本
- 检查其他人是否能够加入（即主机是否正确地将 host_setting.json 中指定的端口进行了端口转发。默认端口是 `1984`）
- 确保你和主机拥有相同的模组
- 如果你认为问题是由模组的略微不同版本引起的，你可以尝试使用相同的场景文件

对于主机：
- 检查你使用的是否是你想要的正确 IPv（客户端也需要被告知使用哪个 IPv）
- 检查你的玩家名称不是 a) 空的，或 b) 不合适的
- 如果使用 IPv6，检查你的路由器是否支持它
- 如果使用 IPv4，不要输入分号（例如 `:1283`），因为这会被解释为 IPv6 地址
- 检查你的防火墙没有阻止 Alice
- 检查你的互联网连接
- 检查主机端口（默认 `1984`）是否在你的机器上正确转发（你可以使用开放端口检查器来分析）
