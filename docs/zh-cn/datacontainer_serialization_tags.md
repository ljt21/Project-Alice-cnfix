## 这是不同数据容器序列化标签的列表，以及对其用途的简短解释

`save` : 带有此标签的属性和对象将被保存到存档文件中，并在多人游戏中通过网络发送给新客户端。

`scenario` : 带有此标签的属性和对象将被保存到场景文件中。这些数据在游戏会话期间应该是恒定的，不会改变。

`local_save` : 带有此标签的属性和对象应该只能在本地访问，不应保存在存档文件中，也不应发送给多人游戏中的客户端。由于 dcon 中目前有一个不正确的 "load_save" 构造，这目前不能正常工作。

`keep_after_state_reload` : 带有此标签的属性和对象被排除在状态重载调用时被重置之外（例如来自多人游戏中的存档重载）。这通常是本地或 UI 内容，对游戏状态本身没有任何影响。

`mp_checksum_excluded` : 带有此标签的属性和对象被排除在 mp_state 校验和之外，该校验和用于确定多人游戏中客户端何时 OOS。这也通常是本地或 UI 内容，但不总是与上一个标签相同的属性。

`mp_data` : 带有此标签的属性和对象在新客户端加入时或在为 OOS 客户端启动重新同步时，在存档之前序列化并发送给客户端。这应该是对发送给客户端很重要的数据，但存储在存档本身中没有意义。

没有任何标签的属性或对象将包含在 mp_state 校验和中，并在状态重置调用时被重置，但不会保存到场景或存档文件中。通常 `fill_unsaved_values()` 函数将在状态重载后填充这些值。
