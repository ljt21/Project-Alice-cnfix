# 如何升级地图

本简短教程介绍如何为现有的 Victoria 2 模组升级地图，使其 (a) 使用正确的投影，以及 (b) 覆盖整个地球。这需要花费多少时间取决于你希望结果有多好；要获得最佳效果，预计需要投入大量时间。在按照这些说明操作时，如果你发现任何不清楚的地方，可能还想参考现有的 HPMGlobe 模组。

## 第一步：准备工作

首先，如果你计划将新地图作为现有模组的子模组制作，请在 `Victoria 2\mod` 目录中创建一个新文件夹。如果不打算这样做，你可以直接在现有模组的目录中进行以下更改。

## 第二步：升级 `provinces.bmp`

本节说明将假设你使用 `paint.net` 来编辑图像。你可以使用任何支持编辑 `png` 文件的程序，但细节会有所不同。你需要在模组的 `map` 目录中创建一个名为 `alice_provinces.png` 的新图像文件。此文件是 `provinces.bmp` 的新等效文件。从 OpenV2Map 模组中的 `map\terrain\colormap_political.dds` 开始。然后，将其垂直翻转，使其看起来正面朝上。接着，在其下方添加第二个图层。大部分工作将在这个第二个图层中完成。你可以选择将其留空开始，或者如果你的模组地图与已升级的地图共享大量省份，也可以选择在该图层中以另一个已升级的地图开始。最后，降低顶层的不透明度，以便你能同时看到它和底层。此时，你应该看到类似以下内容：

![step 1](step1.png)

接下来，你将把正在升级的地图的各个部分复制到第 2 层中，将它们拉伸以适应第 1 层所示的世界。在此过程中，请确保将 `Nearest Neighbor` 设置为采样模式（这可以确保在此过程中不会创建新的像素颜色）。

![step 2](step2.png)

我建议在此过程中使用相对较薄的水平条带，条带两侧具有明显可识别的特征（例如海岸线），从容易处理的大陆开始，然后再处理更难定位的区域（因为你可以使用已放置的区域来引导你）。

![step 3](step3.png)

在此过程中需要注意以下几个主要事项：
- 北美和南美大陆在大多数模组地图中位置太靠北（在此过程中你需要纠正这一点）。
- 东太平洋的大多数岛屿没有以这种方式调整，但可能更容易将所有海域省份与大陆一起移动，然后将岛屿移回它们适当的位置。
- 让大西洋正常工作将是地狱般的体验：祝你好运
- 西伯利亚的最东端高度扭曲，需要大量关注
- 许多岛屿省份与现实相比大得离谱，你需要决定是将它们缩小到现实的一两个像素还是保持原样

一旦地图被填满（必要时从另一个已升级的地图复制极地区域），你将不得不回头纠正海岸、湖泊和其他细节。通常，这意味着使用铅笔工具来纠正省份边界与实际地形特征之间的小错位。

![step 4](step4.png)

（在左侧你可以看到省份延伸得太远进入了海地块——深色区域——而在右侧你可以看到它已经纠正到了正确的海岸线）。使用绘图工具时，请确保将它们设置为锯齿渲染和像素化选择质量——你不希望它们向图像添加新颜色。如果你要在任何时候使用填充工具，还请确保将容差设置为 0。

这一步将花费你大部分的时间。

注意，完成后你需要删除顶层（因为 `png` 文件不支持多个图层），并确保你以 `png` 格式保存/导出最终结果（如果你一直在保存进度，文件当前可能不是 `png` 格式）。

## 第三步：添加新的极地省份

完成省份图像后，你需要添加对应于新极地区域的新省份（如果不这样做，它们将完全显示为水）。

首先复制基础模组的 `map/definitions.csv` 文件到你的新地图目录中。打开该文件并在末尾添加四个新省份。（你可以添加超过 4 个，但 4 是我用来填充新区域的最小数量）。

![definitions](definitions.png)

注意，你的新省份应该在模组之前的最后一个省份之后连续编号。你还应该检查这四个新省份的颜色是否都未被模组中的某个省份占用（如果是，你需要选择一种新颜色并相应地重新着色省份图像）。

接下来，复制模组的 `default.map` 文件。首先，将文件顶部的 `max_provinces = ` 值增加 4。然后，将两个新海洋省份的省份编号（记住，你的编号可能不同）添加到 `sea_starts` 部分，如下图所示。

![default](default.png)

接下来，复制模组的 `region.txt` 文件。我们将把每个新省份添加到仅包含该省份的新区域中，如下所示：

![regions](regions.png)

注意，你需要适当调整省份编号，无论是在大括号内还是在我们给区域命名的名称中（巧妙的是，我们将直接重用我们给省份本身的名称）。

接下来，复制模组的 `adjacencies.csv` 文件，将下图所示的四行添加到文件末尾（同样，省份编号需要相应调整）。注意，这些行在未修改的 Victoria 2 中不会起作用：第二个值为 0 将确保这些省份根本无法进入。

![adjacencies](adjacencies.png)

要为新省份命名，你需要在模组中添加一个名为 `localisation` 的目录。在该目录中，创建一个名为 `openv2names.csv` 的文件。在该文件中，添加以下 4 行（同样，根据需要替换省份编号）：

![openv2names](openv2names.png)

最后，我们需要为这些新省份创建一些历史条目。在模组中创建目录，使你拥有 `history\provinces\other` 路径。在该文件夹中，从 `OpenV2Map` 复制两个省份历史文件，然后用正确的省份编号重命名它们，如下所示。

![history](history.png)

## 第四步：创建 .mod 文件

最后，你需要为地图升级模组创建一个 `.mod` 文件。你应该以 HPM 全球升级文件为模型，其内容如下：

```
name = "HPM Globe Map"
path = "mod/HPMGlobe"
dependencies = { "HPM Alice Patch" "Open V2 Globe Map" }
replace_path = "mod/OpenV2Map/history/provinces/other"
```

将名称更改为你的升级模组的名称，并将路径更改为你模组所在的文件夹。在依赖项中，将 `HPM Alice Patch` 更改为你正在升级的模组的名称（或者如果需要的话，其兼容性补丁——因为兼容性补丁会反过来引入基础模组）。将 `Open V2 Globe Map` 的依赖项保持原样：这允许你重用已为该模组制作的一些纹理，这可以显著减小你的模组大小。同样保持 `replace_path` 不变（这将防止为 open v2 全球地图添加的省份历史文件与事物发生冲突）。
